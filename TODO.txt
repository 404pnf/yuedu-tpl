6. 建议用户信息提交后服务器返回如下格式信息

  {
    success:  'success msg', // 可有可无
    error:    'error msg',  // 可有可无
    warning:  'warning msg' // 可有可无

  }

  其中键代表状态信息，值是状态信息。

  这个方法优于之前的

  {
    code: 'succuess|error'
    msg:  'success msg | error msg'
  }

  因为之前接口定义的两个键实际上是成对出现的。完全可以放在一个键中，如 statude: [code, msg]。
  即使这样该了，还有不好的地方。就是无法简单表示多个状态信息。
  有些状态信息是互斥的，如success和error。有些不是。比如success的时候可以告诉用户有些现在不是必填项的信息如果填写能得到更有针对性的服务。

  新接口很简单呢。某个键值对都是成对出现的状态和信息。有，我就按需显示出来，没有，就什么都不会显示。
